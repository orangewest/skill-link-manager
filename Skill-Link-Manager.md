# 我给自己写了个小工具：一份 skill 同步到所有 AI 编程助手

> 现在免费的 AI 编程助手一抓一大把，自己也攒了不少好用的 skill，可每多用一个工具，就得把同一份技能复制一遍、改一遍。如果你也被这事烦过，这篇可以看看。

## 一、痛点：工具越来越多，技能却要手动复制

这两年免费的 AI 编程助手特别多，QClaw、WorkBuddy、mimocode、opencode 这些，加上 Cursor、Claude Code 之类，基本都免费或者对个人免费，随手就装了好几个，每个都舍不得删。

用得久了，我也顺手攒了一堆技能（skill）：教 AI 怎么规范提交代码、怎么写单测、怎么按我的习惯办事。技能其实就是写给 AI 的一套指令文件夹，一个 `SKILL.md` 加若干脚本和参考资料。

不少工具都认 Claude 那套 skills 格式，各自读一个 `skills/` 目录：

```
~/.qclaw/skills/
~/.workbuddy/skills/
~/.config/mimocode/skills/
~/.config/opencode/skills/
~/.claude/skills/
...
```

问题就出在这：同一份技能，我得往每个工具目录里各塞一份。这些 skill 是我自己一点点调出来的，越攒越值钱，可每多接一个工具，复制粘贴的活就得多来一轮。

平时基本是这样：

- 调好一个 `git-commit` 技能，Ctrl+C、Ctrl+V 到 8 个目录 😮‍💨
- 改了个 prompt 里的错别字，又得挨个目录同步
- 俩月后回头看，8 份文件三个版本，哪个是最新的？没人说得清
- 新装个工具，还得记得把技能再拷一份

难的倒不是写 skill，是让一堆工具共用同一份、还永远是最新的那份。

## 二、市面上其他做 skill 管理的开源工具

我不是头一个碰这问题的人。动手前我看了看别的开源工具，大家出发点都差不多，做法不太一样：

- **Skills Manager**（Tauri 2 + Rust 桌面应用）：最直接的同类。功能很全，标签、Preset 预设、Git 备份、更新检查、自定义工具路径都有。
- **Skiller**（跨平台桌面工具）：强在分发，一份技能能同时发到全局、指定项目、多个工具的组合目标；还接了 NPX / skills.sh 市场，能在线搜了直接装，复制和软链接两种模式都给。
- **AI Skills Manager**（npm CLI）：命令行版，交互式选要注册到哪些工具，适合爱敲终端的人。
- **skillx**（Node/PNPM CLI）：走订阅聚合的路子，用一份 YAML 描述本地源加远程 Git 源，自动克隆、递归找 `SKILL.md`、去重、生成，再链进各个 agent 目录；支持 dry-run、content-hash 去重。

放一张表对比：

| 工具 | 形态 | 同步方式 | 远程订阅 / 市场 | 特点与取舍 |
| --- | --- | --- | --- | --- |
| Skills Manager | 桌面 (Tauri2+Rust) | 软链接 (symlink) | 无（本地） | 功能最全：标签 / Preset / Git 备份 / 更新检查 |
| Skiller | 桌面 | 复制或软链接（双模式） | 集成 NPX / skills.sh 市场 | 强在分发：全局 + 项目级多目标 |
| AI Skills Manager | CLI (npm) | 软链接 | 无（本地） | 轻量命令行，交互式选工具 |
| skillx | CLI (Node) | 生成副本 + 软链 | 强：本地+远程订阅/去重 | 偏订阅聚合 + 生成，配置较重 |
| Skill Link Manager（本文） | 桌面 (Tauri2+Rust) | 目录联接 (junction) / 软链 | 无（聚焦本地链接） | 极简：只做链接，一处维护处处生效 |

这些开源工具都挺成熟，功能也强，远程订阅、技能市场、标签管理、Git 备份、Preset 预设要啥有啥。

我做这个工具没想跟它们拼功能。纯粹是自己用着省心：不追求全覆盖，只把「一份技能、链到所有工具」搞定，剩下的复杂活交给那些专业工具。

## 三、Skill Link Manager 是什么

一个基于 Tauri 2 的跨平台桌面小工具（Windows / macOS / Linux 都能跑），干的事很单一：

> 在一个共享目录里编辑技能，它就会以链接的形式出现在所有 AI 工具里。改一处，所有工具同步。

核心模型很简单：

```
共享技能目录（默认 ~/skills）
├── git-commit/        ← 技能只存这一份
├── code-review/
└── write-tests/
         │
         │  创建链接（不是复制！）
         ▼
~/.claude/skills/git-commit    ─┐
~/.cursor/skills/git-commit     ├─ 全都指向同一份源文件
~/.codex/skills/git-commit      │
~/.config/opencode/skills/...  ─┘
```

技能在硬盘上只有一份实体，所有工具读到的都是它。在共享目录里改一个字，所有工具立刻同步，因为它们本来就读的同一个文件。

## 四、能干啥

- 共享目录里编辑一次，所有已链接的工具立刻能看到，不用再复制粘贴。
- 内置 18+ 主流 AI 工具：opencode、codebuddy、claude、cursor、comate、windsurf、gemini、codex、cline、roo、continue、copilot、kiro…… 首次启动会自动探测你本机装了哪些，帮你预填好。
- 主页用卡片列出每个技能，以及它当前链到了几个工具；点进详情页能针对单个工具目录单独开或者关链接。
- 不想让某个技能进 Cursor？点一下关掉就行，不搞全有或全无。
- 第一次打开有 onboarding，自动检测已装工具、生成默认配置，开箱即用。
- 内置 i18n，中英文一键切换。

## 五、我的取舍：不拼功能，只求自己用着顺手

上面那些工具功能比我这个强太多，我也没想跟它们比全能。我刻意把它做薄，只留自己每天最用得上的几点。

### 1. Windows 用 Junction，普通用户就能建、打开即用

工具在编译期就按平台自动选链接方式。Windows 上用目录联接（Directory Junction），不需要管理员权限，双击打开就能用；macOS / Linux 用系统原生的符号链接，效果一样。

| 平台 | 链接方式 | 关键优势 |
| --- | --- | --- |
| Windows | 目录联接（Junction） | 普通用户权限即可，开箱即用 |
| macOS / Linux | 符号链接（Symlink） | 系统原生软链接 |

双击就能用，不用先去折腾环境。

### 2. 纯链接，不复制、不订阅

Skiller 有复制模式，复制出去的副本改了源也不更新；skillx 偏从远程源生成副本。我的工具只有链接一种形态，技能只存一份：

- 改一处，所有工具同步，不存在版本漂移；
- 硬盘不冗余，一份就是一份；
- 想统一下线某个技能，删源文件就行（工具侧的链接自然失效）。

我不需要花哨的同步引擎，链接够日常用了。

### 3. 轻量、原生、跨平台

基于 Tauri 2，不是 Electron 那种重壳。后端 Rust，前端 React 18 + TypeScript + Tailwind，打包出来体积小、启动快、原生窗口，Windows / macOS / Linux 一套代码全覆盖。比起要配 YAML 源、记一堆命令的订阅型工具，它开箱即用、几乎零配置。

## 六、怎么用

### 第一步：启动后看主页

打开应用，主页会列出共享目录里扫到的所有技能。第一次进来时，它们都还没建立任何链接：

![首次启动后的主页](./img/img1.png)

### 第二步：准备共享目录里的技能

主页里看到的技能，都来自一个共享目录（默认 `~/skills`）。只需要在里面新建文件夹、写好 `SKILL.md` 和配套脚本，应用会自动扫到，不用手动登记。

### 第三步：在详情页建立链接

点进某个技能（比如 `skill1`）进入详情页，可以看到所有已检测到的工具目录，逐个勾选要链到的工具，或者点「应用到全部」一键全开：

![技能详情页 - 逐工具管理链接](./img/img2.png)

链接即时生效。回到主页，卡片上就会显示它的已链接状态：

![建立链接后的主页](./img/img3.png)

### 第四步：按需调整目录配置

想改共享目录位置、增删或勾选工具目录，点右上角「设置」进入目录配置页即可：

![目录配置页](./img/img4.png)

### 第五步：到文件系统里验证

打开 Windows 下某个 AI 工具的技能目录（比如 opencode 的 `.config/opencode/skills`），能看到 `skill1` 以链接（junction）的形式存在：

![Windows opencode 技能目录中的链接](./img/img5.png)

到这里就完成了，打开对应的 AI 工具，技能已经在那儿了。

## 七、适用场景与限制（说点实在的）

- 它只管技能的链接，不接管各工具本身的安装和配置；
- 链接指向共享目录，所以共享目录别随便删或搬，否则所有链接失效；
- 删或改共享目录里的技能会同时影响所有已链接工具，这是「一处维护、处处生效」的一体两面；
- 实际生效取决于该工具是否真的读它的 `skills/` 目录，绝大多数认 Claude 技能格式的工具都没问题；
- 刻意不做远程订阅聚合 / 技能市场搜索（那是 skillx、Skiller 的强项），本工具聚焦本地一份、链接到多处；
- 目前没有后台自动同步，链接的建立和更新在应用里手动点一下就行（本来也是低频操作）。
